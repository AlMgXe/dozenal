tgmconv: A Tutorial

Tgmconv is a broadly applicable unit converter that can convert from U.S. customary, imperial, SI metric, or imperial, into any of the same systems. It works entirely in dozenal, and can work with complex units of arbitrary complexity.

Command-Line Options

tgmconv sports some of the same options as dec and doz and dozdc; specifically, -k for precision and -e to get exponential notation in your output. These work precisely as they do in those other programs, so there's no need to explain them further here.

tgmconv does, however, have three additional command-line options: -i, -o, and the final argument, which is the quantity to be converted.

That final argument is simply a number, and doesn't require much explanation; if you want to convert five seconds into Tims, just use "5" as the final argument. The other two take units as their arguments; -i takes the input unit, and -o takes the output unit. tgmconv expects to see its units in a particular format. I've endeavored to make this format as intuitive as possible; the user will have to judge whether I've succeeded.

Unit Syntax

I've endeavored to make the unit syntax as simple and intuitive as possible; but to make the program generally applicable required a pretty complex system. Fortunately, for most common applications it's very simple, and more complicated requirements can be addressed pretty quickly. Below we treat the syntax for each of the unit systems known by tgmconv, one at a time.

As this is a tutorial, not a manual, I won't be including an exhaustive list of possible units and proper abbreviations here. Users should see the manual for that.

TGM Units

TGM units have a simple syntax; they are simply entered with their canonical TGM abbreviation, which is always a capital letter followed by one or more lowercase letters. So, for example, "Grafut" is abbreviated "Gf", and is entered as such:

-i Gf

TGM also offers an easy way to write when we're raising or lowering a unit by a given power of 10 (the base, twelve), similar to that offered by SI metric with its many prefixes (such as "milli" and "kilo"): this consists of a number prefixed to the unit, superscripted if the power of 10 is positive and subscripted if the power of 10 is negative. This looks like the following normally:

3Gf; 6Gf

Our formatting is more limited in plain text, though, so what we do instead is utilize the symbols ^ for superscripting and _ for subscripting, placed between the digit and the unit, like so:

-i 3^Gf
-o 6_Gf

The above indicates that the input unit is triquaGrafut (3Gf) and the output unit is hexciaGrafut (6Gf).

TGM units can also be multiplied and divided in the same way as SI and customary-imperial units; but since the method is rarely necessary in TGM, with its very complete set of basic units, we'll save the discussion of that for then.

SI Metric Units

Like TGM, each unit in SI metric has a canonical abbreviation; e.g., "s" for seconds, "m" for meters, and "Ci" for curies. However, SI frequently requires compound units, much more often than TGM (in which they are always available but usually optional), so there's a defined syntax for using such units.

Compound units are built by multiplying and dividing simple ones; e.g., the unit of velocity is distance divided by time. In TGM, one can list this as Gf/Tm (Grafut per Tim), but there is also a simple unit, Vlos (Vl). In SI, there is no such simple unit, so the compound unit must be used. So we express division of units with the forward-slash, /. We express multiplication with the asterisk, *. Finally, we also must exponentiate some units; acceleration, for example, is meters per second squared. We do exponentiation simply by suffixing a digit to that unit; e.g., m/s2 for meters per second squared, or meters per square second.

Sometimes quite complex units must be used, combining all of these techniques; e.g., the unit of conductivity, in TGM simply the Eldu, must be expressed as siemen-meters per square meter, S*m/m2.

Any SI unit can also have a prefix indicating the power of ten to which it is being raised or lowered; e.g., the kilometer is the meter raised to the power of 3, while the micrometer is the meter lowered to the power of 6. SI does not, however, permit a simple numerical abbreviation, as TGM does; we must use these alphabetic prefixes. tgmconv embodies these prefixes in the following forms:

The user will note that these are all standard, except for exa, peta, tera, and mega. These had to be different because, when combined with certain units, they conflicted with certain TGM units; for example, the megagram, by standard abbreviation "Mg", conflicts with the TGM Mag (Mg), the unit of force. So these four must be abbreviated in the non-standard, but still intuitive, way shown above; all others can be used as-is.

SI units modified by these prefixes can still be used in compound units.

Customary and Imperial Units

These work more or less the same as metric units, except that some common abbreviations for compound units will not work. For example, "miles per hour" is commonly abbreviated "mph"; tgmconv will not understand this at all. Rather, it must be given as mi/hr. Similar, "psi" will not work for "pounds per square inch"; this unit must be entered as lbf/in2.

Using tgmconv

So putting it all together, we can use tgmconv pretty simply. Let's say we want to convert seconds into Tims:

$> tgmconv -i s -o Tm 1
5;9153

Or perhaps we'd like years convert into quadquaTims:

$> tgmconv -i yr -o 4^Tm 3
13260

So three years is 13260 quadquaTim. Or, perhaps we'd like to see kilometers in quadquaGrafut:

$> tgmconv -i km -o4^Gf "2;6"
0;4X87

So two and a half (2;6) kilometers is 0;4X87 of a quadquaGrafut.

The input unit can be specified without the -i option, if you include it as part of the quantity to be converted. You must then list your final argument as the number, followed by some non-dozenal character (that is, any character that could not be part of a dozenal number), followed by the unit in its usual abbreviation. E.g., our last example could have been entered thus:

$> tgmconv -o4^Gf "2;6:km"
0;4X87

We have used a colon (:) for the non-dozenal character separating the number from the unit; however, any character could have been used as long as it could not be confused for a part of the number.

Note that if you use both this method and the -i option, tgmconv will become very confused, and if it manages to give you an answer at all, it will probably be wrong.

Conclusion

And that is tgmconv, a powerful program for converting between metric systems in dozenal.
